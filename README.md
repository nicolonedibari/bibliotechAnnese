# Il progetto è pensato per essere facilmente riproducibile su qualsiasi macchina senza dover installare manualmente PHP o MySQL, in quanto l’intero ambiente viene avviato tramite Docker Compose. Per eseguire il progetto è necessario avere installato Docker. Dopo aver clonato il repository GitHub e aperto la cartella principale del progetto (quella contenente il file docker-compose.yaml), è sufficiente eseguire il comando docker compose up --build per avviare automaticamente tutti i servizi. La prima esecuzione potrebbe richiedere alcuni minuti perché Docker scarica le immagini necessarie e inizializza il database. Una volta completato l’avvio, l’applicazione è accessibile all’indirizzo http://localhost:9080. È inoltre disponibile phpMyAdmin per la gestione del database all’indirizzo http://localhost:9081 e Mailpit, utilizzato come simulatore SMTP per visualizzare le email di OTP, all’indirizzo http://localhost:9082. Il database viene creato automaticamente al primo avvio tramite uno script SQL caricato nella cartella sql/ ed eseguito dal container MySQL. Sono presenti utenti di test con hash reali delle seguenti password: 1234, bella, ciao e addio. Gli account di esempio sono: mario@mail.com(password: 1234), luca@mail.com(password: bella), anna@mail.com(password: ciao) e admin@mail.com (password: addio). I primi 3 sono utenti-studenti, l'ultimo è studente-amministratore. Il codice sorgente PHP si trova nella cartella src/ ed è montato come volume nel container web; questo permette di modificare i file durante lo sviluppo senza dover ricostruire l’immagine Docker, poiché ogni modifica è immediatamente visibile nel browser. Per arrestare l’applicazione è sufficiente eseguire docker compose down, mentre per rimuovere completamente anche il database si può utilizzare docker compose down -v.
